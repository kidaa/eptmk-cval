NAME

medpost - part of speech tagger for MEDLINE text  

SYNOPSIS

medpost [-xml|-medline|-text|-idtext|-token|-idtoken] [-penn] [-id] [-nosegment] [-titles] [-command] [-version] [file]

 

DESCRIPTION

Medpost is a part of speech tagger for MEDLINE. Text is read from file (or standard input if no file is specified), tokenized, segmented into sentences, tagged with parts of speech, and printed to standard output. The format of the input file defaults to ITAME, with XML and MEDLINE as options. The output tag set is the MedPost tag set, with Penn treebank and SPECIALIST lexicon post-tagger translation options.

 

OPTIONS

-xml
The input text is formatted in the XML format generated by MEDLINE.
-medline
The input text is formatted in the MEDLINE format.
-text
The input text consists of lines of text to be tagged.
-idtext
Same as -text but with an identifier on each line preceding text to be tagged. The identifiers are output as with the -id option.
-token
The input text consists of pre-tokenized text, with tokens separated by white space. Any tags are removed, if they are appended to a token by an underscore.
-idtoken
Same as -idtoken but with an identifier on each line preceding pre-tokenized text to be tagged. The identifiers are output as with the -id option.
-penn
The tagged output will be translated to the Penn treebank tag set.
-id
Each line of output is preceeded by an identifier. Identifiers are constructed from the PMID of the text (or any alpha-numeric identifier given in the .I field) with an additional sentence identifier. The sentence identifier is an A or T, depending on whether the sentence comes from the abstract or title, followed by the sentence number.
-titles
Both titles and abstracts will be tagged. By default, only abstracts are tagged.
-nosegment
Text is tagged without being broken into sentences.
-command
Echo the command line (pipeline) that would be used to invoke the tagger. The commands are not executed.
-version
Print the version date and installation date. No commands are executed.

 

TRANSLATION

Tagged output can be translated to the Penn treebank tag set when it is tagged by using the -penn option. Alternatively, text that has been tagged with MedPost tags and saved to a file can be translated with the perl script.

perl util/tag2penn.perl [-preserve] [ file ... ]

This script can be executed from any directory, just provide the path name to the perl scripts. If file is not explicitly given on the command line, then the standard input is read. If the -preserve option is specified, then any non-tagged text encountered will be printed to the standard output without change.

 

OPERATION

The shell script medpost in the MedPost installation directory provides the recommended usage of the tagger. The script will read text from a file or from standard input for tagging, and tagged output is written to standard output.

There are currently three input options for text: ITAME, XML, and MEDLINE. The XML and MEDLINE formats are the formats produced by MEDLINE when saving search results with one of these display options. (see http://ncbi.nlm.nih.gov). The default format is the ITAME format, and this convenient format may be used when tagging text from non-MEDLINE sources.

 

ITAME FORMAT

The ITAME file format places all text of interest on lines beginning with a period, followed by a field indicator, followed by the text of the field. The text in each field should appear on a single line with no line breaks. The field indicators are

I
identifier
T
title
A
abstract
E
end of record

The M field indicator, from which the ITAME acronym is derived, stands for MeSH, but this field is ignored by the tagger. Furthermore, any line that does not start in this way is ignored.

As an example, the record for the PUBMED abstract with PMID 2359731 would look like

> .I2359731
> .TGiant centrifugal miliaria profunda.
> .AA 4-month-old developed rapidly enlarging ...
> .E

The input file, regardless of its format, may contain arbitrarily many records. The identifiers are used only to construct sentence identifiers (for the -id option) , and whether derived from MEDLINE or not, they need not be unique.

 

INSTALLATION

The script file medpost may be invoked from any directory, or copied to any directory and executed. However, after moving or renaming the install directory, it must be re-installed by typing make or, equivalently, by executing the perl script util/install.perl.

If the installation script fails, the installation can be completed by hand.

1.
Edit the medpost file, which is a shell script, and set the TAGGER_HOME variable to the path name of the MedPost installation. For reference, the install script also updates the IDATE variable to contain the date and time of the installation.
2.
Set the working directory to the src directory and type make. The make file in this directory uses the g++ compiler by default, but the install script will replace this with CC under solaris operating systems. If your system has a different C++ compiler, edit the src/makefile and set the CC variable to the command name.

 

TESTING

Input text has been provided for testing, together with the tagged text that should result. The files test/query.itame, test/query.medline and test/query.xml contain the results of a MEDLINE query, and test/query.tagged is the result of tagging any of these. After successful installation, to execute the test script, change the working directory to the test subdirectory and execute the shell script runtest,

% sh runtest

If any differences are encountered, a message will be displayed, which may indicate an error in the installation.
   

INTERNALS

The medpost script invokes the program util/tokenizer to perform all tokenization and segmentation of input. This program generates commands that are read by the util/tagger program directing it to tag the sentences and print the results. If specified, the output is passed to util/tag2penn.perl to translate the MedPost tags to Penn treebank tag set before printing. These programs are executed by the shell in a pipeline, and each program has been written to process one line at a time as it is read, allowing tagged output may appear before tokenization of the input file has completed.

The lexicon is stored in medpost.lex and the ngram frequencies are stored in medpost.ngrams. In addition, a file medpost.pairs contains some abbreviations that are used by the sentence segmenter.

The tokenizer and tagger are implemented as C++ classes, and these functions can be called directly from a C++ program. See the example files (MPtokEX1.C, MPtokEX2.C, MPtokEX3.C, MPtagEX1.C, and MPtagEX2.C) in the src directory.

 

WORD DATABASE

An ASCII database of the 100,000 most frequent words from MEDLINE is available in the file models/words.db. The words of MEDLINE were mapped to lower case, then all strings of digits were converted to a single digit 1, and all punctuation was changed to the string ``.punctuation.'' These words are then sorted by their descending frequency in MEDLINE and are followed by the part of speech tags, and the number of times each tag is assigned to that word by the tagger. If a tag entry has a ``+'' appearing before or after it, it indicates that the word was tagged as part of a multi-word lexical item.
